# Dino English CRM 第三期需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目标
基于「CRM 与运营平台需求对齐」会议纪要与官方迭代计划，CRM 第三期的核心目标是建设**销售中心**，承接注册线索的分配与人工跟进转化，并接入**外呼系统**，形成「注册 → 分配 → 跟进/外呼 → 付费」的转化闭环。

**核心能力：**
- 销售分配 leads
- 跟进 leads（待领取 → 领取跟进 → 转化付费回流用户中心）
- 接入外呼（点号码拨打 + 通话小结自动归档）
- 通话记录集中管理
- 数据权限隔离

### 1.2 目标用户
- **销售 / 业务跟进**：在销售中心领取「注册+有手机号」的线索，记录跟进、发起外呼，促成付费转化。
- **客服 / 运营**：辅助跟进与查看。
- **管理员**：全盘数据查看与管理。

---

## 2. 业务规则与流转机制

### 2.1 进入 / 离开规则（漏斗口径）
销售中心仅处理具备触达条件（有手机号）且尚未付费的潜在客户线索。

| 场景 | 归属 | 说明 |
| --- | --- | --- |
| **注册（未付费）+ 有手机号** | **销售中心线索** | 自动进入「待领取」线索池 |
| 注册（未付费）+ 无手机号（第三方登录） | 用户中心 | 拿不到手机号，无法电话跟进，仅在用户中心留档 |
| **跟进中转为「已付费」** | **回流用户中心** | 用户状态改写为「付费」，离开销售中心 |
| 已付费 / 付费逾期 | 用户中心 | 全部在用户中心管理 |

### 2.2 跟进进度定义
| 进度 | 含义 |
| --- | --- |
| **待领取** | 线索已进入销售中心，尚无销售认领 |
| **跟进中** | 已被销售领取，正在联系与转化 |
| **暂不跟进** | 暂时搁置（如家长仍在观望）；仍归属该销售 |
| **已付费** | 转化成功，用户状态改写为「付费」，回流用户中心 |

## 4. 页面与数据字典

### 4.1 列表字段（待领取 / 我的跟进）
| 字段 | 说明 |
| --- | --- |
| 用户ID / 姓名 | 复用用户中心字段（本地名优先） |
| 用户类型 | 正式用户 / 测试用户（按规则识别，便于剔除测试线索） |
| 年龄段 / 登录账号 / 业务线 | 复用用户中心字段 |
| 注册渠道 | 展示「业务线 · 渠道」 |
| 渠道来源 | 投放 / KOL 归因标识（如 `KR_META_SUMMER`） |
| 注册时间 | 底层 UTC 存储，按注册国家换算当地时间并标注时区 |
| 跟进进度 | 待领取 / 跟进中 / 暂不跟进（转「已付费」即离开销售中心） |
| 最新备注 / 下次跟进 / 最后更新 / 领取人 | 仅在「我的跟进」列表展示；最新一次跟进快照 |

### 4.2 通话记录字段
| 字段 | 说明 |
| --- | --- |
| 起呼时间 | 发起外呼的时间 |
| 客户 / 手机号 / 业务线 | 被呼叫线索的客户姓名、号码与所属业务线 |
| 通话结果 | 已接通 / 无人接听 |
| 时长 | 通话时长（`mm:ss`）；无人接听为「—」 |
| 本次跟进记录 | 坐席填写的通话小结（必填），同步写入销售跟进记录 |
| 外呼坐席 | 发起外呼的操作人 |

## 5. 关键交互流程

1. **领取线索**：「待领取注册用户」Tab 中，销售按关键字/业务线筛选线索，点「领取跟进」认领 → 自动进入「我的跟进」。
2. **常规跟进**：在「我的跟进」点「更新跟进」→ 选择进度（跟进中 / 暂不跟进 / 已付费）、填下次跟进时间与本次备注（必填）→ 保存写入跟进记录。
3. **外呼跟进**：在「我的跟进」点客户号码旁「外呼」→ 弹窗「正在呼叫 → 通话中」→ 挂断后填通话结果 / 时长 / 通话小结（必填）→ 保存，生成通话记录并归档到跟进记录。
4. **查看历史**：切到「通话记录」Tab 查看全部外呼历史；弹窗底部时间轴亦可查看该线索历次跟进（进度 · 备注 · 时间 · 领取人）。
5. **转化闭环**：若本次跟进选「已付费」→ 用户状态改写为「付费」，该用户离开销售中心，在用户中心可查。

---

## 6. 权限与安全
- **菜单可见性**：销售 / 客服 / 管理员可见且「可操作」，运营为「可查看」，市场投放不可见。
- **数据隔离**：非超管销售仅见本人领取的线索与本人外呼记录；超管/管理员可见全业务线全部线索与通话记录。
