Dino English 运营平台（CRM）产品需求文档

本文档基于「CRM 与运营平台需求对齐」会议纪要，按业务诉求分期落地。一期聚焦基础查询能力（渠道、用户基本信息、订单信息），二期聚焦商业化配置能力（课包、优惠券）。文档内嵌可交互原型，可逐模块体验。

背景与目标

Dino English 面向海外多业务线（韩国 / 沙特 / 马来 / 越南 等）开展投放与运营。当前业务依赖人工散表查询用户与订单、渠道归因口径不统一、促销配置依赖研发。CRM 一期目标是先把「查得到、看得清、能归因」做扎实，让市场与客服同学自助完成日常查询；二期再把商品（课包）与促销（优惠券）配置能力交还运营，减少对研发的依赖。

一期价值

  • 渠道结构标准化、可生成投放 code
  • 用户注册按渠道 code 归因
  • 用户 / 订单一处可查，替代散表

二期价值

  • 课包（商品）运营自助上下架
  • 满减券自助配置发放
  • 促销与商品打通，闭环可用

衡量指标

  • 用户/订单查询自助率 ≥ 90%
  • 渠道归因覆盖率 ≥ 95%
  • 促销配置不再依赖研发

迭代计划 / 路线图

按「先查询、后配置」的节奏分两期交付，先解决业务每天都要用的查询场景，再交付运营自助配置能力。下列时间为相对排期建议（可按实际研发资源调整）。

一期基础查询能力建议 3~4 周

目标：渠道结构标准化并可生码归因；客服 / 运营可自助查询用户基本信息与订单信息，替代线下散表。

🌿
渠道管理 P0
渠道类型 → 一/二/三级渠道树；任意层级生成可复制的渠道 code。
👤
用户基本信息查询 P0
按 学生ID/姓名/手机号 搜索，业务线、状态筛选；可修改基本资料。
🧾
订单信息查询 P0
按 订单ID/学生ID/商品 搜索，订单状态、支付方式筛选。

一期交付物 / 验收

  • 渠道树可增删改、可生码并复制；用户注册渠道 code 与渠道树打通。
  • 用户、订单列表查询、筛选、分页可用；字段口径与本 PRD 数据字典一致。
  • 登录鉴权（@dinoai.ai 邮箱验证码）可用。
二期商业化配置能力建议 3~4 周

目标：把商品（课包）与促销（满减券）的配置能力交还运营，与订单 / 商品打通，形成「配置 → 发放 → 核销」闭环。

📦
课包管理 P1
新增/编辑/详情/上下架；业务线、币种、价格、有效期。
🎟️
优惠券管理 P1
满减券配置：发放规则、使用规则（按课包 ID 绑定可用商品）、满减权益；支持编辑/延长/停止发放。

二期交付物 / 验收

  • 课包可上下架，下架后不可被券绑定（前置校验）。
  • 满减券可生成、可调整可用商品、可延长领取时间、可停止发放（二次确认）。
  • 券与课包通过 课包ID 关联，币种与业务线一致。
后续能力增强（暂不在本次范围）规划中

会议纪要中提出但不在一 / 二期承诺范围的需求，记录在册、后续评估。

  • 用户标签体系 & 黑名单识别
  • 角色权限 / 数据权限（市场、销售、客服分权）
  • 邀约场景 & 提报成单率统计
  • 第三方整合：Breeze 消息推送、Apespire 流量进度
  • 商品品类丰富度扩展、活跃测试
  • 非手机号注册支持、注册平台多样化

1.1 渠道管理 一期 P0

维护标准化的渠道分级结构，并在任意层级生成实际投放使用的渠道 code，作为用户注册归因的唯一口径。

用户故事

作为市场投放同学，我希望按「渠道类型 → 一级 → 二级 → 三级」维护渠道，并为具体投放位生成唯一 code，以便用户注册时能准确归因到来源。

功能清单

功能点说明优先级
新增渠道类型如 自然流量 / landingpage / KOL，作为树的根P0
分级渠道维护在类型下逐级新增 一/二/三级渠道（如 国家 → 媒体 → 达人）P0
生成 / 查看渠道 code任意层级生成唯一 code，可一键复制；已生成则展示并支持复制P0
重命名 / 删除删除会级联删除其下级渠道与 code（二次确认）P1

数据字典

字段说明示例 / 规则
渠道类型渠道树根节点自然流量 / landingpage / KOL
渠道层级一级 / 二级 / 三级一期支持至三级，结构可扩展
渠道 code实际投放/归因使用的唯一标识natural_kr_aso_4f9k2a（类型前缀 + 随机串）

关键交互流程

1
点击「新增渠道类型」→ 输入名称（如 KOL）。
2
在类型/渠道节点 hover，点「+」逐级新增下级渠道。
3
在目标层级点闪电图标「生成 code」→ 弹窗展示 code → 复制。
4
该 code 即用于投放链接；用户由此注册时写入 渠道 code，在用户中心可见。
🔗
归因闭环：渠道 code 是「渠道管理」与「用户中心 / 订单」的连接键。务必与投放侧统一生码与回传口径。

1.2 用户基本信息查询 一期 P0

客服 / 运营按多条件检索学生（用户），查看其基本信息、注册归因与状态，并可维护可编辑的基础资料。

用户故事

作为客服同学，我希望输入学生ID / 姓名 / 手机号即可查到该用户，并看到他的业务线、注册渠道、注册时间与当前状态，以便快速响应咨询。

列表 + 分页展示基本信息字段，支持分页与每页条数P0

2.1 课包管理 二期 P1

运营自助维护课包（商品）：新增、编辑、查看详情、上下架，作为订单与优惠券绑定的商品基础。

用户故事

作为运营同学，我希望自助创建课包（设置业务线、价格、币种、有效期）并控制上下架，无需每次找研发。

功能清单

功能点说明优先级
新增课包业务线 / 名称 / 币种 / 价格 / 有效期；默认上架P1
编辑课包修改上述字段P1
详情查看含创建人、创建时间、状态P1
上 / 下架二次确认；下架后不可被新券绑定P1
搜索 / 筛选课包ID / 名称、业务线P1

数据字典

字段说明规则
课包ID唯一标识PKG1001
业务线归属业务线决定可选币种
币种本地币种 + USD见附录映射
价格 / 有效期售价 + 起止时间有效期为时间段
课包状态上架 / 下架下架不可售卖/绑券
🔗
课包 课包ID 是二期优惠券「可用商品」的绑定键；优惠券通过课包 ID 回车搜索添加可用商品。

2.2 优惠券管理 二期 P1

运营自助配置满减券：定义发放/领取规则、使用规则（绑定可用课包）、满减权益，并支持编辑、延长、停止发放。

用户故事

作为运营同学，我希望为某业务线生成满减券，设置发放数量、领取/使用有效期、可用课包与「满 X 减 Y」权益，并能在活动中途调整或停止发放。

功能清单

功能点说明优先级
生成券（选业务线）先选业务类型 → 进入满减券表单（一期券类型仅满减券）P1
发放及领取规则发放数量、领取有效期P1
使用规则使用有效期 + 可用商品（按课包ID搜索添加）P1
权益规则满减：满 X 减 Y（抵扣 ≤ 门槛）P1
编辑可用商品 / 延长领取时间列表内操作P1
停止发放二次确认 → 领取状态置「已结束」P1

数据字典

字段说明
券ID / 券码唯一标识 + 12 位券码
业务线 / 币种生成时锁定，与课包一致
券总量 / 剩余数量发放总量与库存
领取有效期 / 使用有效期两段独立时间
可用商品课包列表（ID / 用户侧名称 / 现价）
满减规则满 thresholdAmount 减 deductAmount
领取状态已生效 / 已结束

关键业务规则

  • 抵扣金额不得大于门槛金额（表单校验）。
  • 可用商品必须为已存在的课包，按课包 ID 校验，不存在则报错。
  • 停止发放后不可继续领取；领取截止时间可向后延长以恢复「已生效」。
  • 券币种、业务线与所绑课包须一致（前置约束）。
