# Dino English 运营平台（CRM）· 迭代计划 + 交互式 PRD

基于「CRM 与运营平台需求对齐」会议纪要与官方迭代计划整理的**分期迭代计划**与**可交互 PRD**。文档按业务诉求分四期落地，并内嵌真实在线原型可逐模块体验。

> **官方迭代计划：** 一期 = 查看用户 / 订单基本信息（当前目标 **7/24 上午**）；二期 = 配置渠道、商品(SKU)、生成投放链接（目标 **8/7 上线**）；三期 = 销售分配 leads、跟进 leads（**接入外呼**，目标 **8/21 上线**）；四期 = 本地化配置自助（暂无上线时间）。
>
> **一期排期变更（因 App 内优化需求高优而延期）：** ① 第一次排期 **7/9** → ② 第二次排期 **7/16** → ③ 第三次排期（当前目标）**7/20 下班前提测 → 测试约 3 天 → 预计 7/24 上午上线**。

## 产品定位

**一句话定位：** Dino English 运营平台（CRM）是面向海外多业务线的**一体化运营中台**——以「用户 · 渠道 · 商品」三大主数据为核心，打通「渠道归因 → 商品/促销配置 → 投放落地 → 用户触达 → 服务履约」的运营闭环，让市场、运营、客服等**非研发角色自助完成日常运营**，把对研发的依赖降到最低。

随业务成熟度四阶段演进：

| 阶段 | 定位关键词 | 核心能力 | 主要服务对象 |
| --- | --- | --- | --- |
| 一期（当前目标 7/24 上午） | 数据可查 · **看得清** | 用户中心（应用商店来源 + 国家粗归因）、订单中心查询、多语言 | 运营 · 销售 |
| 二期 | 归因 + 自助配置 · **配得了** | 渠道生码归因（含渠道参数）、用户中心升级为「业务线 · 渠道」精准归因、商品包(SKU)、满减券（一券多码按 KOL 结算）、落地页一键生成投放链接、角色权限 | 运营 · 市场投放 |
| 三期（目标 8/21） | 销售 leads · **够得着** | 销售中心（分配 leads / 跟进 leads + **接入外呼 / 通话归档**，注册线索转化闭环） | 销售 · 客服 · 运营 |
| 四期 | 本地化自助 + 触达服务 · **改得动** | 将现有本地化文案 / 物料配置平台迁入 CRM，业务按业务线 / 语言自助维护文案与物料；单点触达用户（push/站内信/短信/EDM）、增加服务节点（生命周期 SOP、任务流转、召回/续费/关怀）；**手动给单用户加会员（待评估）** | 运营 · 市场投放 · 本地化 · 客服 · 学习服务 |

## 目标用户（内部 B 端角色）

> **当前使用方（暂定）：运营 + 销售。** 一期主要由运营与销售使用（用户中心 / 订单中心查询）；下表为平台完整角色画像，其余为后续 / 潜在角色，随能力上线逐步引入。

| 角色 | 核心诉求 | 高频模块 |
| --- | --- | --- |
| 🛠️ 运营 / 商业化 `当前` | 快速查到用户/订单、看清状态与有效期；二期商品/促销自助配置、不依赖研发；后期触达与服务编排 | 用户中心 · 订单中心 ·（二期）商品包/优惠券/落地页 ·（后期）触达/服务节点 |
| 💼 销售 / 业务跟进 `当前` | 查到用户与订单、看清付费状态与有效期；三期在销售中心领取「注册+有手机号」线索并记录跟进 / 外呼、促成付费 | 用户中心 · 订单中心 ·（三期）销售中心（分配/跟进 leads + 外呼） |
| 📣 市场投放 / 增长 `后续` | 渠道结构清晰、可生码、投放链接一键可得、归因可追 | 渠道管理 · 落地页 |
| 🎧 客服 / 用户支持 `后续` | 快速查到用户与订单、看清状态与有效期；后期单点触达答疑 | 用户中心 · 订单中心 ·（后期）触达/服务节点 |
| ⚙️ 管理员 / 系统配置 `后续` | 账号鉴权（限 `@dinoai.ai`）、业务线/模版配置、角色权限 | 系统配置 · 角色权限 |
| 🧑‍🏫 学习服务 / 班主任（后期） | 按生命周期节点拿到待办、对单用户触达与跟进、记录服务结果 | （后期）服务节点 · 单点触达 · 用户中心 |

## 打开方式

直接用浏览器打开 `index.html` 即可（无需构建）。也可部署到 GitHub Pages 后在线访问。

## 分期范围

| 期次 | 模块 | 核心能力 | 优先级 |
| --- | --- | --- | --- |
| **过渡期**（一期上线前） | 线下导表触达 | 产研导表 → 业务老师录入 Lark 表分配 leads → 销售在外呼公司平台外呼；人工过渡方案，系统就绪后由三期销售中心替代 | 现状 |
| **一期 · 基础查询**（7/9→7/16→7/24 上午） | 用户中心（一期） | 多条件查询（用户ID/姓名/登录账号）+ 用户类型（正式/测试）/注册渠道（App Store/Google Play）/国家/状态筛选 + 到期时间/最近修改人 + 修改资料 + 修改历史（字段变更前后追溯）+ 试听报告临时入口；用户状态精简为 注册/付费/付费逾期 | P0 |
| | 订单中心 | 多条件查询 + 用户类型/国家筛选 + 有效期到期时间 | P0 |
| | 平台能力 | 多语言（英/韩/阿/中，默认英文） | P0 |
| **二期 · 营销中心**（目标 8/7） | 渠道管理 | 业务线 → 类型 → 多级渠道 + 生成渠道 code + 渠道参数（带入落地页链接） | P1 |
| | 商品包管理（SKU） | 增改 / 上下架 | P1 |
| | 优惠券管理 | 满减券配置 + 一券多优惠码（按 KOL）独立统计用于结算 | P1 |
| | 落地页管理 | 一键关联渠道 + 商品包 + 优惠码 + 渠道参数，按业务线模版生成投放链接 | P1 |
| **二期 · 用户与权限**（目标 8/7） | 用户中心 · 二期 | 升级为「业务线 · 渠道 · 渠道 code」精准归因 + 业务线全局筛选 + 用户详情页（基本信息 + 课程信息：上课记录 / Trial & Lesson Report / 回放） | P1 |
| | 系统配置 · 角色权限 | RBAC 角色权限矩阵（无/只读/可操作）+ 成员账号 + 业务线数据权限 + 操作日志 | P1 |
| **三期 · 销售**（目标 8/21） | 销售中心 | 销售分配 leads + 跟进 leads → 待领取 → 领取跟进 → 转化付费回流用户中心；**外呼（点号码拨打 + 通话小结自动归档）+ 通话记录**；数据权限隔离 | P1 |
| **四期 · 本地化 + 触达 + 加会员** | 本地化文案 / 物料平台 | 现有本地化配置平台迁入 CRM，业务自助维护各业务线/语言的文案与物料 | P2 |
| | 单点触达 / 服务节点 | push/站内信/短信/EDM + 生命周期服务节点（SOP / 任务流转 / 召回续费关怀） | P2 |
| | 手动加会员（待评估） | 单用户手动加有效期，替代找产研改库；高风险，权限开放对象与风控待评估 | 评估中 |
| 后续 | 标签 · 黑名单 · 邀约/提报 · 第三方整合 | 记录在册，后续评估 | P2+ |
