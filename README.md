# Dino English 运营平台（CRM）· 迭代计划 + 交互式 PRD

基于「CRM 与运营平台需求对齐」会议纪要整理的**分期迭代计划**与**可交互 PRD**。文档按业务诉求分两期落地，并内嵌真实在线原型可逐模块体验。

## 产品定位

**一句话定位：** Dino English 运营平台（CRM）是面向海外多业务线的**一体化运营中台**——以「用户 · 渠道 · 商品」三大主数据为核心，打通「渠道归因 → 商品/促销配置 → 投放落地 → 用户触达 → 服务履约」的运营闭环，让市场、运营、客服等**非研发角色自助完成日常运营**，把对研发的依赖降到最低。

随业务成熟度三阶段演进：

| 阶段 | 定位关键词 | 核心能力 | 主要服务对象 |
| --- | --- | --- | --- |
| 一期 | 数据可查 · **看得清** | 渠道生码归因、用户/订单查询、多语言 + 业务线全局筛选 | 客服 · 市场 |
| 二期 | 自助配置 · **配得了** | 商品包、满减券（一券多码按 KOL 结算）、落地页一键生成 | 运营 · 市场投放 |
| 后期 | 触达服务 · **够得着** | 单点触达用户（push/站内信/短信/EDM/外呼）、增加服务节点（生命周期 SOP、任务流转、召回/续费/关怀） | 运营 · 客服 · 学习服务 |

## 目标用户（内部 B 端角色）

| 角色 | 核心诉求 | 高频模块 |
| --- | --- | --- |
| 📣 市场投放 / 增长 | 渠道结构清晰、可生码、投放链接一键可得、归因可追 | 渠道管理 · 落地页 |
| 🛠️ 运营 / 商业化 | 商品/促销自助配置、不依赖研发；后期触达与服务编排 | 商品包 · 优惠券 · 落地页 ·（后期）触达/服务节点 |
| 🎧 客服 / 用户支持 | 快速查到用户与订单、看清状态与有效期；后期单点触达答疑 | 用户中心 · 订单中心 ·（后期）触达/服务节点 |
| ⚙️ 管理员 / 系统配置 | 账号鉴权（限 `@dinoai.ai`）、业务线/模版配置；后续角色权限 | 全局配置 ·（后续）权限管理 |
| 🧑‍🏫 学习服务 / 班主任（后期） | 按生命周期节点拿到待办、对单用户触达与跟进、记录服务结果 | （后期）服务节点 · 单点触达 · 用户中心 |

## 打开方式

直接用浏览器打开 `index.html` 即可（无需构建）。也可部署到 GitHub Pages 后在线访问。

## 分期范围

| 期次 | 模块 | 核心能力 | 优先级 |
| --- | --- | --- | --- |
| **一期 · 基础查询** | 渠道管理 | 业务线 → 类型 → 多级渠道 + 生成渠道 code | P0 |
| | 用户中心 · 基本信息查询 | 多条件查询（用户ID/姓名/登录账号）+ 到期时间/最近修改人 + 修改资料 | P0 |
| | 订单中心 · 订单信息查询 | 多条件查询 + 业务线筛选 + 有效期到期时间 | P0 |
| | 平台能力 | 多语言（中/英/韩/阿）+ 业务线全局筛选 | P0 |
| **二期 · 商业化配置** | 商品包管理 | 增改 / 详情 / 上下架 | P1 |
| | 优惠券管理 | 满减券配置 + 一券多优惠码（按 KOL）独立统计用于结算 | P1 |
| | 落地页管理 | 一键关联渠道 + 商品包 + 优惠码，按业务线模版生成投放链接 | P1 |
| | 系统配置 · 角色权限 | RBAC 角色权限矩阵（无/只读/可操作）+ 成员账号 + 业务线数据权限 + 操作日志 | P1 |
| 后续 | 标签 · 黑名单 · 邀约/提报 · 第三方整合 · 单点触达/服务节点 | 记录在册，后续评估 | P2+ |
