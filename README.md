# Dino English 运营平台（CRM）· 迭代计划 + 交互式 PRD

基于「CRM 与运营平台需求对齐」会议纪要整理的**分期迭代计划**与**可交互 PRD**。文档按业务诉求分两期落地，并内嵌真实在线原型可逐模块体验。

## 打开方式

直接用浏览器打开 `index.html` 即可（无需构建）。也可部署到 GitHub Pages 后在线访问。

## 分期范围

| 期次 | 模块 | 核心能力 | 优先级 |
| --- | --- | --- | --- |
| **一期 · 基础查询** | 渠道管理 | 业务线 → 类型 → 多级渠道 + 生成渠道 code | P0 |
| | 用户中心 · 基本信息查询 | 多条件查询（用户ID/姓名/登录账号）+ 到期时间/最近修改人 + 修改资料 | P0 |
| | 订单中心 · 订单信息查询 | 多条件查询 + 业务线筛选 + 有效期到期时间 | P0 |
| | 平台能力 | 多语言（中/英/韩/阿）+ 业务线全局筛选 | P0 |
| **二期 · 商业化配置** | 商品包管理 | 增改 / 详情 / 上下架 | P1 |
| | 优惠券管理 | 满减券配置 + 一券多优惠码（按 KOL）独立统计用于结算 | P1 |
| | 落地页管理 | 一键关联渠道 + 商品包 + 优惠码，按业务线模版生成投放链接 | P1 |
| 后续 | 标签 · 黑名单 · 权限 · 邀约/提报 · 第三方整合 | 记录在册，后续评估 | P2+ |

## 文档结构（index.html）

- **文档概览**：背景、目标、分期一览
- **迭代计划 / 路线图**：两期目标、交付物、验收、里程碑与依赖
- **需求拆解**：会议纪要逐条映射到分期与模块
- **一期 PRD**：渠道管理、用户基本信息查询、订单信息查询（含数据字典 + 内嵌原型）
- **二期 PRD**：商品包管理、优惠券管理（满减券 + 多优惠码结算）、落地页管理（含业务规则 + 内嵌原型）
- **交互原型预览**：6 模块切换体验
- **附录**：业务线/币种/国家码映射、状态枚举、术语表
